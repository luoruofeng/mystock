# 涨跌排行页面功能说明

## 功能概述

新增涨跌排行页面，展示沪市（SS）、深市（SZ）和港股（HK）的涨跌排行榜。

## 访问地址

```
http://localhost:8000/rankings
```

## 页面特性

### 1. 市场筛选
- **全部市场**：同时展示三个市场的涨跌排行
- **沪市 (SS)**：仅展示沪市股票
- **深市 (SZ)**：仅展示深市股票
- **港股 (HK)**：仅展示港股股票

### 2. 显示数量
支持选择显示前 10、20、30、50 名股票

### 3. 排行榜内容

每个市场展示两个榜单：

#### 涨幅榜
- 显示涨幅最大的股票
- 按涨跌幅降序排列
- 红色高亮显示涨幅

#### 跌幅榜
- 显示跌幅最大的股票
- 按涨跌幅升序排列
- 绿色高亮显示跌幅

### 4. 显示信息

每个股票显示以下信息：
- 排名序号
- 股票代码（可点击查看详情）
- 股票名称
- 最新价
- 涨跌幅（百分比）

## 数据说明

### 数据来源
- 从 `stock_history` 表获取最新交易日的收盘价
- 从 `stocks` 表获取股票基本信息

### 计算方法
```
涨跌幅 = ((最新收盘价 - 前一日收盘价) / 前一日收盘价) × 100%
```

### 颜色说明
- **红色**：上涨（符合中国市场习惯）
- **绿色**：下跌（符合中国市场习惯）

## 使用方法

### 1. 访问页面
在导航栏点击"涨跌排行"按钮，或直接访问 `/rankings` 路径

### 2. 筛选市场
在页面顶部的下拉菜单中选择市场：
- 选择"全部市场"可查看三个市场的排行
- 选择具体市场可聚焦查看该市场排行

### 3. 调整显示数量
选择需要显示的股票数量（10-50名）

### 4. 查看详情
点击股票代码可跳转到股票详情页面查看更多信息

## 数据更新

涨跌排行数据依赖于股票历史数据表，建议定期更新：

```bash
# 更新股票历史数据（示例）
python scripts/update_stock_history.py
```

## 技术实现

### 后端路由
- 文件：`app/routers/pages.py`
- 路由：`/rankings`
- 参数：
  - `market`：市场筛选（默认 ALL）
  - `limit`：显示数量（默认 10）

### 前端页面
- 文件：`app/templates/rankings.html`
- 框架：Bootstrap 5 + Jinja2
- 特性：响应式布局，支持移动端访问

### 导航菜单
- 文件：`app/templates/base.html`
- 位置：导航栏"板块行情"之后

## 注意事项

1. **数据依赖**
   - 需要有股票历史数据才能显示涨跌排行
   - 至少需要两天的历史数据才能计算涨跌幅

2. **实时性**
   - 页面显示的是最新交易日的数据
   - 非交易时间数据不会更新

3. **性能优化**
   - 对于大量数据，建议添加缓存机制
   - 可考虑添加分页功能
